Android Studio

1. 问题

1.1. EditText.getText().toString() 获取不到值

EditText.getText().toString() 获取不到值通常是因为在获取值的时候 EditText 控件还未完成初始化或者是在获取值的时机上出现了问题，例如在 EditText 的 onClick 事件中获取值，而此时用户可能还没有输入任何内容。

解决方法

确保 EditText 在获取值的时候已经完成了初始化。

如果是在按钮点击事件中获取值，确保在用户有足够时间输入内容后再获取值。

检查是否在 EditText 输入内容前调用了 getText().toString() 方法。

1.2. 向下一个activity发送数据

1.2.1. 方法1：

当前activity：

//单独可发送一条字符串数据
intent.putExtra("StringMessage","其他数据");

下一个activity：

Intent i=getIntent();
String StringMessage = i.getStringExtra("StringMessage");

1.2.2. 方法2：

当前activity：

//创建一个包裹,发送数据
Bundle bundle =new Bundle();
bundle.putString("StringMessage","其他数据");
bundle.putString("TestMessage","测试数据2");
intent.putExtras(bundle);

下一个activity：

Bundle bundle =getIntent().getExtras();
String StringMessage =bundle.getString("StringMessage");
String TestMessage =bundle.getString("TestMessage");

1.3. Android studio 安装app的图标显示2个

检查app的AndroidManifest.xml文件中，是否存在多个

<intent-filter>
    <action android:name="android.intent.action.MAIN" />
    <category android:name="android.intent.category.LAUNCHER" />
</intent-filter>

1.4. 使用WebView加载非https链接无显示

从Android 9.0（API级别28）开始，默认情况下禁用明文支持。因此http的url均无法在webview中加载，Android9.0对未加密的流量不再信任，添加了新的限制。

解决办法

  1. 创建文件 res/xml/network_security_config.xml

    其中将192.168.1.100替换为你自己的域名，根域名即可，includeSubdomains=”true” 就包含了二级域名。

  2. 修改 AndroidManifest.xml，application节加入

    android:usesCleartextTraffic=”true”

    android:networkSecurityConfig=”@xml/network_security_config”

  3. 检查网络访问权限，修改 AndroidManifest.xml，application节上方加入
